1. Introduction

Student Schedule Planner (SSP) is a desktop application aimed at helping students manage their schedules effectively. SSP is optimised for users who prefer to type rather than click. In fact, if you type efficiently, this application will be much faster than regular to-do applications. Move on to the Section 2, “Quick Start” to get started.

2. Quick Start

  1. Install Java 9 if you do not have it. It is available here.

    The application may not run correctly on Java 10 or Java 11.

  2. Download the latest studentscheduleplanner.jar here.

  3. Copy the file to the folder that you want to use as the home folder for your planner.

  4. Double-click the file to start the app. The GUI should appear in a few seconds.

Figure 1. The menu that you should see once you open the application.

  1. Type a command in the command box and press Enter to execute it.
    e.g. typing help and pressing Enter will open the help window.

  2. Refer to Section 3, “Features” for details of each command.

3. Features

Command Format

  • Words in UPPER_CASE are the parameters to be supplied by the user e.g. in add n/NAME, NAME is a parameter which can be used as add n/do CS2100 tutorial.

  • Items in square brackets are optional e.g n/[NAME] t/[TAG] can be used as n/do CS2100 tutorial t/CS2100 or as n/do CS2100 tutorial.

  • Items with …​ after them can be used multiple times including zero times e.g. t/[TAG]…​ can be used as   (i.e. 0 times), t/tutorial, t/tutorial t/CS2100 etc.

  • Parameters can be in any order e.g. if the command specifies n/[NAME] d/[DEADLINE], d/[DEADLINE] n/[NAME] is also acceptable.

All tasks will be automatically loaded in the order of deadlines, one that due earlier will appear before one that due later. If two tasks have the same deadline, they will be arranged according to priority, one that has higher priority will appear in front of one that has lower priority. 3 is the highest priority, while 1 is the lowest.

3.1. Adding a Task

Adds a task to the schedule planner.

add n/[NAME] p/[PRIORITYLEVEL] t/[TAG] d/[DEADLINE] v/[VENUE]

Example:

Input: add n/exam p/3 t/CS3241 d/101018 v/mpsh1

Outcome: Adds a task named exam with priority level 3, tag CS3241 and deadline of 10th October 2018, venue at mpsh1.

The priority level must be a number between 1 to 3, where 3 denotes highest priority.

The name, priority level and deadline fields are compulsory and required for every task. Tags are optional.

3.2. Adding a Recurring Task

Adds copies of the same task to a specified day in the week, for a given number of weeks.

repeat r/[REPEATS] n/[NAME] p/[PRIORITYLEVEL] t/[TAG] d/[DEADLINE] v/[VENUE]

Example:

Input: add r/3 n/Lecture p/3 t/CS3241 d/101018 v/mpsh1

Outcome: Adds 3 tasks named Lecture with priority level 3, tag CS3241, venue at mpsh1, and deadline of 10th October 2018, 17th October 2018, and 24th October 2018.

The number of repeats should be an integer that is greater or equals to 1. Setting the repeat as 1 has the same effect as using the add command.

Use this command to schedule tasks that you carry out regularly.
The maximum number fo repetitions is 15.
Due to how the Date class was implemented, there is an unintended behavior. Any task whose deadline goes beyond the year 2099 will have the deadline set to the 21st century instead. For example, a task created by the AddRepeatCommand that has the deadline 1st January 2101 will instead have the deadline incorrectly set to 1st January 2001.

3.4. Editing a Task

Edits an existing task in the schedule planner.

edit [INDEX] n/[NAME] p/[PRIORITYLEVEL] t/[TAGS]…​ d/[DEADLINE]

Example:

Input: edit 3 p/1 d/121019
Outcome: Priority of existing task with index 3 to 1, and its deadline is changed to 12th October 2019 .

The INDEX refers to the index number shown in the displayed task list. Refer to the task according to the index number. The INDEX must be a positive integer 1, 2, 3, …​

At least one of the optional fields must be provided. Existing values of provided field will be updated to the input values.

Existing tags of a task will be removed when its tag is edited. If input is edit 3 t/project, then the resulting tag(s) of task 3 will be project only.
To remove all tags of an existing task, you can just type t/. If input is task 3 t/, all the tags of the task with INDEX 3 will be removed.
Use list task, listday or listweek to find out the index of the task to be modified.
For example, you can use listday to list the tasks due today, and choose the index of the task you want to edit.

3.5. Finding Tasks by Name

Finds tasks whose names match with given keyword.

find [NAME]

Example:

input: find tutorial
output: All tasks with "tutorial" in their names are listed.

The search is case insensitive. For example, apples and Apples both return the same tasks.

The order of the keywords does not matter. For example, apples buy and buy apples both return the same tasks.

You can include multiple keywords when using find. find CS3230 CS5229 will find all tasks with name containing CS3230 or CS5229.

3.6. Filtering Tasks by Tag

Filters and lists tasks with tags matching those entered by the user. User may input more than 1 tag.

filter [TAG] [TAG2] …​

Example:

  • filter lecture
    output: All tasks with the tag tutorial are listed.

  • filter lecture 2100
    output: All tasks with the tag either tutorial, 2100, or both tutorial and 2100, are listed.

The search is case insensitive. e.g apples matches Apples.

You can include multiple tags when using filter. filter tutorial quiz project will return all tasks with tag matching with at least one of the provided tags in input.

3.7. Deleting a Task

Deletes the specified task from the schedule planner forever.

delete INDEX

Example:

input: delete 3
output: Task with index 3 delete from schedule planner.

The INDEX refers to the index number shown in the displayed task list. Refer to the task according to the index number. The INDEX must be a positive integer 1, 2, 3, …​

Use the list command to find out the index of the task to be deleted.

tag::archivetask[] === Archiving a Task

Archives the specified task from the schedule planner. This is different from deleting a task as it is equivalent marking the task as completed. You may also view the archived tasks later.

archive INDEX

Example:

input: archive 4
output: Task with index 4 is removed from the current list and saved in the archived list.

The INDEX refers to the index number shown in the displayed task list. Refer to the task according to the index number. The INDEX must be a positive integer 1, 2, 3, …​

Use the list command to find out the index of the task to be archived. end::archivetask[]

output: Shows a list of tasks from 021118 to end of the month which is 301118.

3.12. Generating Weeks in the Academic Calendar

Generates the entire academic calendar with description, based on the first academic day and stores it in storage.

Currently, the generated academic calendar is based on the academic calender of National University of Singapore (NUS). It may not be compatible with academic calendars from other institutions.

Whenever the application is launched within the academic calendar’s dates, the application title will append that particular week’s description to the title of the app.

firstday DDMMYY

Example:

input: firstday 130818
output: When the user launches the application from E.g 221018 to 281018 (Week 10 for NUS academic year 18/19 semester 1), the application title will append "Week 10" to the title of the app.

There would not be any visible changes upon using the command. The reflection would only occur after relaunching the application.

The DDMMYY refers to the date format of day, month and year. It must complies the following three rules in order:

  1. The DDMMYY must only be one set of value such as 130818. Value such as 130818 200818 or 130818 20 will be rejected as they are considered as more than one set of date.

  2. The DDMMYY must be a valid date within 21st century.

  3. The DDMMYY must be a Monday.

3.15. Displaying Today’s Progress

Shows the percentage of tasks that have been archived for the day in the command result box. It also lists the uncompleted tasks for today. The progress bar is also displayed at the bottom left of the window.

progresstoday

3.16. Displaying This Week’s Progress

Shows the percentage of tasks that have been archived from today to the nearest Sunday in the command result box. It also lists the uncompleted tasks from today until the nearest Sunday. The progress bar is also displayed at the bottom right of the window.

progressweek

3.17. Listing Used Commands

Shows a list of all the commands that you have entered in reverse chronological order.

history

Pressing the ↑ and ↓ arrows will display the previous and next input respectively in the command box.

3.18. Undoing Previous Command

Restores the schedule planner to the state before the previous undoable command was executed.

undo

Undoable commands: those commands that modify the schedule planner’s content (add, delete, edit and clear).
Use the history command to decide if you want to undo the previous undoable command.

3.19. Redoing the Previous Undo Command

Reverses the most recent undo command.

redo

Examples:

delete 1
clear
undo (reverses the clear command)
undo (reverses the delete 1 command)
redo (reapplies the delete 1 command)
redo (reapplies the clear command)

3.20. Clearing Schedule Planner

Clears all entries from the schedule planner.

clear

3.21. Exiting the App

Exits the application.

exit

3.22. Saving the Data

Data in the Student Schedule Planner is saved in the hard disk automatically after any command that changes the data.
There is no need to save manually.

5. Possible Questions

Q: How do I transfer my data to another Computer?
A: Install the app in the other computer and overwrite the empty data file it creates with the file that contains the data of your previous Schedule planner folder.
